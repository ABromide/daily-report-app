# Daily Report App

Agent Research Briefing App is a local-first macOS and static web workstation for
reviewing AI research signals collected by Codex automations.

The project has two user-facing surfaces:

- `web/`: an Astro static site that reads public fixture/data manifests and can be deployed to GitHub Pages.
- `mac/`: a SwiftUI macOS app that imports the same public data contract into a responsive local cache.

The data plane is intentionally separate from the application code:

- `main` stores source code, schemas, workflows, documentation, and committed fixtures.
- `data` stores append-only public data generated by automation.

## Quick Start

```bash
uv run pytest
uv run pyright
npm --prefix web install
npm --prefix web run build
npm --prefix web run test:e2e
swift build --package-path mac
swift run --package-path mac DailyReportSmoke
```

## Design Direction

This is an Agent intelligence workstation, not an article wall. The interface is
dense, searchable, and evidence-oriented: left navigation, high-density feed,
reader/inspector detail, KPI strip, source/run health, and explicit sync status.

The visual baseline follows the `kami` editorial system:

- parchment background `#f5f4ed`
- ivory panels `#faf9f5`
- ink-blue accent `#1B365D`
- warm neutral text and borders
- restrained depth and strong reading rhythm

## Implementation Status

This repository starts with fixture data first. Real hourly Codex automation is
enabled only after schema, web, macOS, CI, Pages, and local sync checks pass.
